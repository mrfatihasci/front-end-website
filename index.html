GitHub

Setup

  • sudo add-apt-repository ppa:git-core/ppa
  • sudo apt update
  • sudo apt install git
  • git --version // see if installed

Settings

  • git config --global user.name "Your Name"
  • git config --global user.email "yourname@example.com"
  • git config --global init.defaultBranch main //setting default branch as main since new repos in github has default branch as main instead of master
  • Next, google how to create ssh key, how to link ssh key to your github account by yourself

Work in a repo

  • git clone 'SSH key' //insert your SSH key from remote github repo which is something like: git@github.com:{your_username}/{repo_name}
  • git add //stage changes you have made
  • git commit -m "fsdfsdfsdf" // write a commit message in the dummy text
  • git push // git push origin main // git push origin branch_name

Branch

  • Creating & pushing branch

    • git branch ------- // see current branches.
    • git branch branch_name ---------- // create a develop branch named branch_name.
    • git checkout branch_name --------- // change(checkout) to branch_name.
    • git checkout -b branch_name -------- // use #2 and #3 in a single command.
    • git push origin branch_name -------- // push branch & now u can see this branch in GitHub repo(after add & commit).

  • Merging branches

    • git merge branch_name -------- // take all commited changes in branch_name & add to the branch you are currently on.
    • git push origin main -------- // push branch(main / master ).

  • Deleting unnecessary merged/unmerged branches

    • git branch -d branch_name ----------- // delete branch_name from your local repo if it is merged.
    • git branch -D branch_name ------------ // delete branch_name from your local repo if it is not merged.
    • git push origin --delete branch_name ---------- // delete branch_name from your remote repo.

  • Undo Add/Commit when not pushed

    • git reset --soft HEAD1 ----------- // does uncommit the last 1 commit yet does not unstage (u can unstage changes by following "git status"s instruction).
    • git reset --mixed HEAD1 ------------ // ..
    • git reset --hard HEAD1 ---------- // ..

You can refer to Pro GIT for further git and github related questions.

Html

Most programming languages(python, c++, java) require a compiler to compile your "source code", however you dont need a compiler to run your HTML CSS JAVASCRIPT codes. Create a folder and open it in vscode. Add an index.html in the folder.

The basic structure of html is shown below in which DOCTYPE element tells browser this is an html file, the html element includes two element; head and body.

  • Head is where you define the title and links.
  • Body is where you include elements to be seen on your page

Now having the basic structure (boilerplate) of an html file, we can learn which html elements to insert in it.

Assignment 1

  • Create a repository in Github and use 'git clone {ssh_key_for_repo}' to copy the remote repo to your local repo
  • Create an index.html file within the repo
  • Add one element within the body of index.html.
  • Save the file, and right click index.html, open it in your broser to see the changes you just have made.
  • Use 'git add index.html', 'git commit -m "your_comit_message"', 'git push origin main' to push your changes from local repo to remote github repository.
  • Use 'git checkout -b branch_name' and create new branch // remember this 'checkout -b' creates new branch and checkouts into it.
  • Make sure you are within the new created branch by 'git branch'
  • Use 'code .' to open repo in vscode, add a bunch of more html elements to index.html in vscode, save the file
  • Use 'git add index.html', 'git commit -m "your_comit_message"', 'git push origin branch_name' to push your changes in new branch to remote github repository.
  • checkout to main branch and use 'git merge branch_name' to merge new branch to main branch
  • use 'git push origin main' to push local main branch to remote main branch repo.
  • you way want to delete branch_name from your local repo by 'git branch -d branch_name'
  • you may want to delete branch_name from your remote repo by 'git push origin --delete branch_name'
  • On your github account, go to your repo, go to setting > pages > branch and select 'main' instead of 'none' to serve your page. Now your page is available on the web
